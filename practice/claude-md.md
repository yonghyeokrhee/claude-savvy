# CLAUDE.md 만들기 실습

CLAUDE.md는 Claude가 프로젝트를 시작할 때 자동으로 읽는 파일이다.
한 번 잘 써두면 매번 설명하지 않아도 Claude가 맥락을 알고 일한다.

## 실습 시나리오

MOP(AI 광고 최적화 서비스) 담당 마케터가 Claude와 함께 일한다고 가정한다.
CLAUDE.md 없이 물어보면 Claude는 MOP가 뭔지 모른다. 있으면 바로 맥락을 파악한다.

## 실습 순서

### 1단계 — 새 폴더 만들고 Claude 시작

터미널에서:

```bash
mkdir mop-project
cd mop-project
claude
```

Claude에게 물어본다:

```
CTR이 갑자기 떨어졌어. 뭘 확인해야 해?
```

Claude는 CTR이 뭔지, 어떤 서비스인지 모르므로 일반적인 답변만 한다.

### 2단계 — CLAUDE.md 작성

같은 폴더에 `CLAUDE.md` 파일을 만든다.

```markdown
# MOP 프로젝트

MOP는 LG CNS의 AI 광고 최적화 서비스다.
검색광고와 쇼핑검색광고를 자동으로 최적화한다.

## 주요 용어

- CTR (클릭률): 광고 노출 대비 클릭 비율. 높을수록 좋음.
- ROAS: 광고비 대비 매출. 500%면 1원 쓰고 5원 번 것.
- 입찰가: 광고 상위 노출을 위해 지불하는 금액. MOP가 자동 조정.

## 사용자

광고주 담당자와 대행사 AE가 주 사용자다.

## 답변 규칙

- 마케터 대상이므로 개발 용어 사용 금지
- 수치 예시는 실제 광고 업계 기준으로
```

### 3단계 — 차이 확인

Claude 새 세션을 열고 같은 질문을 한다:

```
CTR이 갑자기 떨어졌어. 뭘 확인해야 해?
```

이번엔 Claude가 MOP 광고 플랫폼 맥락에서, 마케터 언어로 답변한다.

## 핵심 포인트

| | CLAUDE.md 없을 때 | CLAUDE.md 있을 때 |
|---|---|---|
| 서비스 인식 | 모름 | MOP 광고 플랫폼으로 인식 |
| 답변 수준 | 일반적 설명 | 실무 맥락에 맞는 답변 |
| 매번 설명? | 필요 | 불필요 |

## 잘 쓰는 팁

- **서비스 설명**: 내가 어떤 일을 하는 사람인지
- **주요 용어**: 자주 쓰는 단어와 뜻
- **답변 규칙**: 톤, 길이, 금지 표현
- **자주 쓰는 명령어**: 반복 작업이 있으면 적어두기
