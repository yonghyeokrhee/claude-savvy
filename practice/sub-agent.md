# Sub-Agent 실습

Sub-Agent가 실제로 어떻게 동작하는지 MOP 광고 분석 시나리오로 확인합니다.

## 준비

`practice/` 디렉토리에 아래 파일 3개가 있어야 합니다:

```
practice/
└── data/
    ├── campaign-search.md    # 검색광고 현황
    ├── campaign-shopping.md  # 쇼핑검색광고 현황
    └── budget-usage.md       # 예산 집행 현황
```

`practice/` 에서 Claude를 시작합니다:

```bash
cd practice
claude
```

---

## 실습 1 — Sub-Agent 없이 순차 처리

먼저 파일 하나씩 분석을 요청합니다:

```
campaign-search.md 파일을 읽고 문제 캠페인이 뭔지 알려줘
```

Claude가 파일 1개를 읽고 답합니다. 왼쪽 도구 목록에서 `Read` 1번 실행되는 걸 확인합니다.

---

## 실습 2 — Sub-Agent 병렬 처리

이번엔 3개 파일을 한 번에 요청합니다:

```
data 폴더에 있는 검색광고, 쇼핑검색광고, 예산 현황 파일을 모두 분석해서
2월 종합 성과 보고서를 만들어줘. 잘 되고 있는 것과 개선이 필요한 것을 구분해서 정리해줘.
```

이번엔 왼쪽 도구 목록에서 `Read`가 **거의 동시에 3번** 실행되는 걸 확인합니다.

> **이게 Sub-Agent입니다.** 3개 파일 분석을 각각 독립된 인스턴스에 맡겨 동시에 처리하고, 결과를 취합해서 하나의 보고서로 돌려줍니다.

---

## 실습 결과 비교

| | 실습 1 (순차) | 실습 2 (병렬) |
|---|---|---|
| 파일 읽기 | 1개씩 순서대로 | 3개 동시 |
| 처리 방식 | 단일 Claude | Sub-Agent 3개 |
| 속도 | 느림 | 빠름 |
| 적합한 상황 | 결과가 다음 단계에 필요할 때 | 서로 독립적인 작업일 때 |

---

## 언제 Sub-Agent가 자동으로 생성되나?

Claude가 요청을 받으면 내부적으로 판단합니다:

- **"이 작업들이 서로 연관이 있나?"** → 없으면 병렬로 분리
- **"컨텍스트가 오염될 만큼 데이터가 많나?"** → 많으면 Sub-Agent로 격리

사용자가 직접 지시할 필요 없이 Claude가 알아서 결정합니다.

---

## 더 큰 규모로 해보기

파일이 많을수록 병렬 처리의 효과가 커집니다. 아래 프롬프트를 시도해봅니다:

```
practice 폴더 전체를 파악하고, 어떤 실습이 준비되어 있는지 목록으로 정리해줘
```

Claude가 폴더 내 모든 파일을 동시에 탐색하는 걸 관찰합니다.
