# Workflow 실습 — MOP 월간 리뷰

`practice/` 폴더에서 5단계 Workflow를 직접 실행해봅니다.
순차 처리, 병렬 처리, 확인 포인트를 한 번에 체험합니다.

---

## 전체 흐름

```
1단계: 데이터 파악 (순차)
    └── data 폴더의 파일이 몇 개인지, 어떤 내용인지 파악

2단계: 채널별 분석 (병렬)
    ├── 검색광고 성과 분석
    └── 쇼핑광고 성과 분석

3단계: [확인 포인트] 분석 결과 검토 후 계속 진행 여부 결정

4단계: 보고서 작성 (순차)
    └── 분석 결과를 취합해 report-monthly.md 생성

5단계: 액션 아이템 도출
    └── 다음 달 즉시 실행할 3가지 제안
```

---

## 실행

`practice/` 에서 Claude를 시작합니다:

```bash
cd practice
claude
```

아래 프롬프트를 그대로 입력합니다:

```
다음 순서로 2월 MOP 광고 성과 리뷰를 진행해줘:

1단계: data 폴더에 어떤 파일이 있는지 먼저 파악해줘.

2단계: campaign-search.md와 campaign-shopping.md를 동시에 읽고
       각 채널의 잘 된 점과 문제점을 채널별로 정리해줘.

3단계: 분석 결과 요약을 나에게 먼저 보여주고, 계속 진행할지 확인해줘.

4단계: 확인 후 report-monthly.md 파일에 종합 보고서를 작성해줘.
       형식: 핵심 요약 / 우수 캠페인 / 개선 필요 캠페인 / 예산 제안

5단계: 다음 달에 바로 실행할 수 있는 액션 3가지를 보고서 하단에 추가해줘.
```

---

## 관찰 포인트

| 단계 | 확인할 것 |
|---|---|
| 1단계 | `Glob` 도구로 폴더 탐색 |
| 2단계 | `Read`가 2번 동시에 실행되는지 (Sub-Agent 병렬) |
| 3단계 | Claude가 스스로 멈추고 확인을 기다리는지 |
| 4단계 | `Write`로 `report-monthly.md` 생성 |
| 5단계 | 앞 단계 내용을 바탕으로 맥락 있는 제안이 나오는지 |

---

## 왜 단계를 나누는가?

"2월 광고 분석해서 보고서 만들어줘" 한 줄로도 요청할 수 있습니다.
하지만 단계를 명시하면:

- **3단계 확인 포인트**에서 방향이 맞는지 중간 점검 가능
- 분석과 작성이 분리되어 결과 품질이 높아짐
- 어느 단계에서 문제가 생겼는지 바로 파악 가능
