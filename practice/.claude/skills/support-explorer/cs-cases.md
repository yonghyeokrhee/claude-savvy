# MOP CS 대응 레퍼런스

실제 고객 문의 사례와 모범 답안을 정리한 내부 참고 문서.

## 케이스 1: 신규 광고그룹 입찰최적화 오류

### 고객 문의 유형
신규 그룹 라이브 직후 노출/유입 모수 부족으로 입찰 최적화 오류가 발생하는 경우.
기존 운영 데이터를 덤핑해서 최적화할 수 있는지 문의.

### 핵심 원인
- MOP 입찰최적화 조건: **최근 2주 실적 충족** 필요
- 신규 그룹은 조건 미충족 → 네이버 **추천입찰가** 기반 입찰로 자동 전환됨
- 최적화 화면 비고란에 '최적화에러' 표기되지만, 입찰 자체는 정상 진행됨

### 모범 답안 요점

**단기 조치 (라이브 후 ~2주):**
- 추천입찰가 기반 입찰 유지 (하루 1회 입찰, 입찰가 다소 높게 형성될 수 있음)
- 주요 키워드는 수기 관리 병행 권장
- '최적화에러' 표기는 정상 — 고객에게 사전 안내 필요

**2주 경과 후:**
- 최적화 조건 충족 시점부터 알고리즘 기반 입찰로 전환

**클러스터링 권장:**
- 유입 볼륨이 저조한 소규모 그룹(03, 04 등)은 볼륨이 큰 그룹(01, 02)과 묶어서 최적화
- 묶어야 알고리즘 학습이 원활하게 진행됨

**데이터 덤핑 요청에 대한 답변:**
- **불가** — MOP는 광고그룹 단위로 학습하며, 신규 광고그룹 ID가 생성되면 학습이 처음부터 다시 시작됨
- 기존 그룹 내 변동(키워드 추가·삭제 등)은 기존 학습 데이터가 유지되지만, 그룹 자체를 새로 만들면 학습 초기화

### 답변 시 주의사항
- '최적화에러' UI 표기에 대해 고객이 혼선을 겪지 않도록 선제 안내할 것
- 데이터 덤핑 불가 이유를 "광고그룹 ID 단위 학습" 구조로 명확히 설명할 것
- 클러스터링 제안은 유입 볼륨 데이터를 확인한 뒤 구체적인 그룹명과 함께 제시할 것
