# Skills

## Skills이란?

**Skill**은 Claude Code에서 `/skill-name` 형태로 호출하는 **미리 정의된 프롬프트 워크플로우**입니다.

사용자가 `/commit`이라고 입력하면, Claude Code는 해당 Skill을 자동으로 확장하여 완전한 프롬프트로 실행합니다. 복잡한 지시를 매번 타이핑하지 않아도 되는 **단축키** 같은 개념입니다.

## 어떻게 동작하나요?

```
사용자: /commit
    ↓
Skill 파일 로드 → 전체 프롬프트로 확장
    ↓
Claude가 프롬프트를 실행
```

Skill은 단순한 명령어가 아니라, 특정 작업을 위해 잘 설계된 **프롬프트 템플릿**입니다. 호출 시 현재 컨텍스트(파일 변경 내역, 프로젝트 정보 등)가 함께 전달됩니다.

## 기본 제공 Skills

Claude Code는 자주 쓰는 작업을 위한 Skills를 기본으로 제공합니다:

| Skill | 설명 |
|---|---|
| `/commit` | 변경 내역을 분석해 커밋 메시지 작성 |
| `/help` | Claude Code 사용법 안내 |
| `/review` | 코드 리뷰 요청 |

## 커스텀 Skills 만들기

프로젝트 또는 전역에 나만의 Skill을 정의할 수 있습니다.

**저장 위치:**
- 전역: `~/.claude/skills/my-skill.md`
- 프로젝트: `.claude/skills/my-skill.md`

**예시 — `/daily` Skill 만들기:**

`.claude/skills/daily.md` 파일을 생성합니다:

```markdown
# Daily Standup

현재 git 변경 내역을 보고 다음을 작성해줘:
1. 어제 한 작업 (최근 커밋 기준)
2. 오늘 할 작업 (현재 수정 중인 파일 기준)
3. 블로커 여부
```

이제 터미널에서 `/daily`를 입력하면 Claude가 이 프롬프트를 실행합니다.

## Skills vs 일반 프롬프트

| 구분 | 일반 프롬프트 | Skill |
|---|---|---|
| 재사용 | 매번 타이핑 필요 | `/skill-name` 한 번 |
| 팀 공유 | 어려움 | `.claude/skills/` 폴더로 공유 |
| 일관성 | 사람마다 다름 | 동일한 프롬프트 보장 |
| 관리 | 없음 | 버전 관리 가능 |

## 언제 Skill을 써야 할까?

- 같은 작업을 반복적으로 요청할 때
- 팀원과 동일한 워크플로우를 공유하고 싶을 때
- 프롬프트가 길고 복잡해서 외우기 어려울 때
- 프로젝트 고유의 절차(배포 체크리스트, 코드 리뷰 기준 등)를 자동화할 때
