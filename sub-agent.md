# Sub-Agent

## Sub-Agent이란?

**Sub-Agent**는 Claude Code가 복잡한 작업을 처리할 때 **별도의 독립적인 Claude 인스턴스를 생성**하여 병렬로 실행하는 기능입니다.

메인 Claude가 오케스트레이터(지휘자) 역할을 하고, Sub-Agent들이 각자 맡은 작업을 독립적으로 수행합니다.

## 왜 필요한가?

단일 Claude 인스턴스로 큰 작업을 처리하면 두 가지 문제가 생깁니다:

1. **컨텍스트 오염** — 여러 파일과 결과물이 쌓이면서 Claude의 집중도가 떨어짐
2. **순차 처리** — 독립적인 작업도 하나씩 처리하여 느림

Sub-Agent는 이 두 문제를 동시에 해결합니다.

## 동작 방식

```
메인 Claude (오케스트레이터)
    ├── Sub-Agent 1: API 명세 분석
    ├── Sub-Agent 2: DB 스키마 설계
    └── Sub-Agent 3: 테스트 케이스 작성
         ↓ (모두 완료 후)
    결과 취합 → 최종 구현
```

각 Sub-Agent는:
- **독립된 컨텍스트 윈도우**를 가짐
- 작업 완료 후 결과만 메인 Claude에 반환
- 서로의 작업에 영향을 주지 않음

## 언제 Sub-Agent가 생성되나?

Claude Code가 자동으로 판단하여 Sub-Agent를 생성합니다. `Agent` 도구를 내부적으로 사용합니다:

- 서로 **의존성 없는 작업**이 여러 개일 때
- **넓은 코드베이스 탐색**이 필요할 때 (메인 컨텍스트 보호)
- 장시간 실행될 수 있는 **독립 리서치 작업**

## 실제 예시

```
사용자: "우리 프로젝트의 보안 취약점을 전체적으로 점검해줘"
```

메인 Claude가 작업을 분리:

| Sub-Agent | 담당 |
|---|---|
| Agent 1 | SQL Injection 패턴 검색 |
| Agent 2 | 인증/인가 로직 분석 |
| Agent 3 | 외부 의존성 취약점 확인 |
| Agent 4 | 환경변수 노출 여부 검사 |

네 개가 동시에 실행되고, 결과를 취합하여 종합 보고서 작성.

## Sub-Agent vs 메인 Claude

| 구분 | 메인 Claude | Sub-Agent |
|---|---|---|
| 역할 | 작업 분배 및 취합 | 단일 작업 집중 실행 |
| 컨텍스트 | 전체 대화 기억 | 할당된 작업만 기억 |
| 수명 | 세션 동안 유지 | 작업 완료 후 종료 |
| 생성 주체 | 사용자 | 메인 Claude 자동 생성 |

## 주의사항

- Sub-Agent 실행은 **추가 API 비용**이 발생합니다
- 각 Sub-Agent는 독립된 컨텍스트를 가지므로 **메인 대화 내용을 모릅니다**
- 결과는 메인 Claude가 요약하여 전달받으므로, 세부 중간 과정은 보이지 않을 수 있습니다
