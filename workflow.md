# Workflow

## Workflow란?

**Workflow**는 여러 단계의 작업을 **순서대로 또는 병렬로 연결**하여 복잡한 목표를 달성하는 실행 흐름입니다.

단순히 "이거 해줘" 한 마디로는 처리하기 어려운 작업을, 논리적인 단계로 나누어 안정적으로 완료하는 방식입니다.

## Workflow가 필요한 이유

```
나쁜 예: "우리 서비스에 결제 기능 추가해줘"
    → Claude가 무엇을 어떻게 해야 할지 모호

좋은 예: Workflow로 단계 정의
    1. 요구사항 분석
    2. DB 스키마 설계
    3. API 엔드포인트 구현
    4. 테스트 작성
    5. 문서 업데이트
```

각 단계의 출력이 다음 단계의 입력이 되면서, 일관성 있는 결과를 만들어냅니다.

## Workflow의 두 가지 패턴

### 1. 순차 실행 (Sequential)

이전 단계 결과가 다음 단계에 필요한 경우:

```
분석 → 설계 → 구현 → 테스트 → 배포
```

각 단계가 완료되어야 다음 단계를 시작합니다.

### 2. 병렬 실행 (Parallel)

서로 독립적인 작업을 동시에 처리하는 경우:

```
           ┌── 프론트엔드 구현
분석 완료 ──┼── 백엔드 구현    → 통합 테스트
           └── DB 마이그레이션
```

Sub-Agent를 활용하여 병렬로 실행합니다.

## Claude Code에서 Workflow 설계하기

복잡한 작업을 요청할 때 단계를 명시적으로 지시하면 훨씬 좋은 결과를 얻을 수 있습니다:

```
"다음 순서로 진행해줘:
1. 먼저 기존 auth 코드를 읽고 구조 파악
2. JWT 토큰 방식으로 리팩토링 계획 작성
3. 계획 확인 후 구현 시작
4. 구현 완료 후 테스트 실행"
```

단계 사이에 **확인 포인트**를 넣으면 예상치 못한 방향으로 흘러가는 것을 방지합니다.

## CLAUDE.md로 Workflow 고정하기

프로젝트에서 반복되는 Workflow를 `CLAUDE.md`에 정의하면 매번 설명하지 않아도 됩니다:

```markdown
## 기능 개발 Workflow
새 기능을 개발할 때는 항상 이 순서를 따를 것:
1. `docs/` 폴더의 관련 스펙 문서 먼저 확인
2. 영향받는 기존 코드 파악
3. 테스트 파일 먼저 작성 (TDD)
4. 구현
5. `npm test` 실행하여 전체 테스트 통과 확인
```

## Human-in-the-Loop

긴 Workflow에서 중요한 결정 지점에 사람의 확인을 넣는 패턴입니다:

```
분석 완료 → [사용자 확인] → 구현 시작 → [사용자 확인] → 배포
```

Claude Code는 작업 중 불확실한 상황을 만나면 멈추고 질문합니다. 이를 적극 활용하세요:

```
"각 단계 시작 전에 계획을 먼저 알려주고, 내가 확인하면 진행해줘"
```

## 실전 팁

- **작업이 클수록 단계를 더 잘게 쪼개세요** — 한 단계에서 실패해도 처음부터 다시 하지 않아도 됩니다
- **중간 결과물을 파일로 저장하게 하세요** — 컨텍스트가 초기화되어도 이어서 진행 가능합니다
- **각 단계에 성공 기준을 명시하세요** — "테스트가 통과하면 다음 단계로" 같은 조건을 추가하세요
