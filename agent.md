# Claude Code 완전 정복 — AI 페어 프로그래머와 함께 개발하기

> **대상 독자:** Claude Code를 처음 접하는 개발자부터, 에이전트 워크플로우를 실무에 도입하려는 팀까지.
> **목표:** Claude Code의 작동 원리를 이해하고, 실전에서 바로 쓸 수 있는 패턴을 익힌다.

---

## 1장. Claude Code란 무엇인가

### 1.1 터미널 안의 AI 동료

Claude Code는 Anthropic이 공식 제공하는 CLI(Command-Line Interface) 기반 AI 코딩 도구다. 채팅 창이 아니라 **터미널과 코드베이스 안에서** 직접 동작한다는 점이 핵심이다.

일반적인 AI 채팅 도구와 다른 점:

| 구분 | 채팅형 AI | Claude Code |
|---|---|---|
| 코드 접근 | 복사·붙여넣기 | 파일 직접 읽기/쓰기 |
| 명령 실행 | 불가 | 셸 명령 실행 가능 |
| 프로젝트 맥락 | 매번 설명 필요 | CLAUDE.md로 영속 유지 |
| 결과 반영 | 수동 적용 | 자동 적용 (승인 후) |

### 1.2 에이전트로서의 Claude Code

Claude Code는 단순한 코드 생성기가 아니라 **에이전트(Agent)**다. 에이전트란:

- 목표를 받아 스스로 계획을 세우고
- 도구(파일 읽기, 명령 실행, 웹 검색 등)를 선택해 사용하며
- 결과를 확인하고 다음 행동을 결정하는

자율적으로 작동하는 시스템이다.

```
사용자 요청 → Claude가 계획 수립 → 도구 실행 → 결과 확인 → 완료 or 재시도
```

## 2장. 설치와 첫 실행

### 2.1 설치

Node.js 18 이상이 필요하다.

```bash
npm install -g @anthropic-ai/claude-code
```

### 2.2 인증

```bash
claude
```

첫 실행 시 Anthropic 계정으로 브라우저 인증이 진행된다. 완료하면 터미널로 돌아와 바로 사용할 수 있다.

### 2.3 프로젝트에서 시작하기

```bash
cd my-project
claude
```

또는 바로 작업 지시:

```bash
claude "이 프로젝트가 어떤 구조인지 설명해줘"
```

## 3장. 핵심 도구 — Claude가 사용하는 무기들

Claude Code는 내부적으로 다음 도구들을 자동으로 선택해 사용한다. 이 도구들이 무엇인지 알면 Claude에게 더 정확한 요청을 할 수 있다.

| 도구 | 역할 | 예시 상황 |
|---|---|---|
| **Read** | 파일 내용 읽기 | 코드 분석, 버그 파악 |
| **Write** | 파일 생성/덮어쓰기 | 새 파일 작성 |
| **Edit** | 기존 파일 부분 수정 | 함수 수정, 리팩토링 |
| **Bash** | 셸 명령 실행 | 테스트 실행, 빌드 |
| **Glob** | 파일 패턴 검색 | `*.py` 파일 모두 찾기 |
| **Grep** | 파일 내용 검색 | 특정 함수 사용처 찾기 |
| **WebSearch** | 웹 검색 | 라이브러리 문서 조회 |
| **WebFetch** | URL 내용 가져오기 | 공식 문서 참조 |

### 3.1 도구 승인 모드

Claude가 파일을 쓰거나 명령을 실행하려 할 때 승인을 요청한다. 세 가지 모드가 있다:

- **수동 승인 (기본값)** — 각 작업마다 확인. 안전하지만 느림.
- **자동 승인** — 묻지 않고 실행. 빠르지만 주의 필요.
- **읽기 전용** — 파일 읽기만 허용.

> **팁:** 처음에는 수동 승인 모드로 시작하고, Claude의 작업 패턴에 익숙해진 후에 자동 승인을 선택적으로 허용하라.

## 4장. CLAUDE.md — 프로젝트 헌법

`CLAUDE.md`는 프로젝트 루트에 두는 마크다운 파일로, **Claude가 세션을 시작할 때마다 자동으로 읽는 지침서**다.

### 4.1 기본 구조

```markdown
# 프로젝트명

## 기술 스택
- Python 3.12, FastAPI, PostgreSQL 16
- 패키지 매니저: uv

## 코딩 컨벤션
- 변수명: snake_case
- 클래스명: PascalCase
- 모든 함수에 타입 힌트 필수
- docstring은 Google 스타일

## 테스트
- 테스트 실행: `pytest tests/ -v`
- 커버리지 확인: `pytest --cov=src`

## 중요 파일
- `src/main.py` — 앱 진입점
- `src/config.py` — 환경 변수 설정
- `src/models/` — DB 모델
- `src/routes/` — API 엔드포인트

## 금지 사항
- `git push --force` 사용 금지
- `.env` 파일 절대 커밋 금지
```

### 4.2 CLAUDE.md를 잘 쓰는 법

**넣어야 할 것:**
- 스택과 버전 (Claude가 맞는 문법을 쓰도록)
- 팀 컨벤션 (네이밍, 포매팅 규칙)
- 테스트/빌드 명령어
- 자주 헷갈리는 아키텍처 결정 이유

**넣지 말아야 할 것:**
- 너무 긴 설명 (200줄 이상이면 압축)
- 현재 작업 중인 태스크 (대화창에서 말하면 됨)
- 비밀키, 패스워드

## 5장. 에이전트 모드 — 큰 작업 맡기기

### 5.1 에이전트 모드란

단일 요청·응답이 아니라, Claude가 **여러 단계를 자율적으로 실행**해서 복잡한 목표를 달성하는 방식이다.

```
"새 결제 모듈 만들어줘"
  ↓
1. 기존 코드 구조 파악 (Read, Glob)
2. 설계 방향 제안
3. 모델 파일 생성 (Write)
4. 라우트 파일 생성 (Write)
5. 테스트 실행 (Bash)
6. 실패 시 수정 (Edit)
7. 최종 확인
```

### 5.2 효과적인 에이전트 지시 방법

**명확한 목표 + 제약 조건을 함께 줘라:**

```
❌ "결제 기능 만들어줘"

✅ "Stripe를 사용해서 결제 기능을 추가해줘.
   - src/routes/payments.py에 엔드포인트 추가
   - 기존 User 모델과 연결
   - 실패 시 적절한 HTTP 오류 반환
   - 테스트는 tests/test_payments.py에 작성"
```

**체크포인트를 설정하라:**

큰 작업은 단계별로 나눠서 각 단계마다 결과를 확인한다.

```
1단계: "DB 스키마만 먼저 설계해줘"
2단계: "그 스키마로 SQLAlchemy 모델 만들어줘"
3단계: "CRUD 엔드포인트 추가해줘"
4단계: "테스트 작성해줘"
```

### 5.3 서브에이전트 패턴

Claude Code는 복잡한 작업을 수행할 때 내부적으로 **병렬 서브에이전트**를 활용할 수 있다. 예를 들어 "프로젝트 전체를 분석해줘" 같은 요청은:

- 서브에이전트 A: 프론트엔드 코드 분석
- 서브에이전트 B: 백엔드 코드 분석
- 서브에이전트 C: 의존성 및 설정 분석

를 동시에 실행해 빠르게 결과를 종합한다.

## 6장. 실전 프롬프트 패턴

### 6.1 코드베이스 탐색

```
프로젝트 구조를 파악하고, 핵심 모듈 3개와 각각의 역할을 설명해줘.
```

```
src/auth.py에서 JWT 토큰 검증 로직이 어떻게 동작하는지
단계별로 추적해서 설명해줘.
```

### 6.2 버그 수정

```
tests/test_user.py의 test_login 테스트가 실패하고 있어.
테스트를 실행하고, 실패 원인을 찾아서 수정해줘.
함수 시그니처는 바꾸지 마.
```

### 6.3 기능 추가

```
/api/posts POST 엔드포인트에 입력 검증 추가해줘.
- title: 필수, 1~100자
- content: 필수, 최소 10자
- tags: 선택, 배열, 최대 5개
검증 실패 시 422 상태코드와 명확한 오류 메시지 반환.
```

### 6.4 코드 리뷰

```
src/payments.py를 리뷰해줘.
보안 문제, 엣지 케이스, 성능 이슈를 심각도 순으로 나열해줘.
```

### 6.5 리팩토링

```
src/utils.py의 process_data 함수가 너무 길어.
단일 책임 원칙에 따라 분리해줘. 기존 테스트가 모두 통과해야 해.
```

### 6.6 문서화

```
src/routes/ 디렉토리의 모든 엔드포인트를 읽고
OpenAPI 스펙 형식으로 API 문서를 작성해줘.
```

## 7장. 슬래시 명령어 레퍼런스

세션 내에서 `/`로 시작하는 명령어를 사용할 수 있다.

| 명령어 | 설명 |
|---|---|
| `/help` | 사용 가능한 명령어 목록 |
| `/clear` | 대화 컨텍스트 초기화 (파일은 유지) |
| `/cost` | 현재 세션 토큰 사용량 확인 |
| `/exit` | 세션 종료 |
| `/compact` | 대화 내용을 요약해서 컨텍스트 절약 |

## 8장. Git과 함께 쓰기

### 8.1 안전망으로서의 Git

Claude에게 대규모 변경을 맡기기 전, 반드시 현재 상태를 커밋해두자.

```bash
git add -A && git commit -m "chore: checkpoint before refactor"
```

이후 변경 사항 전체를 확인:

```bash
git diff HEAD
```

문제가 생기면:

```bash
git reset --hard HEAD
```

### 8.2 Claude에게 커밋 맡기기

```
변경된 파일들을 git에 커밋해줘.
커밋 메시지는 Conventional Commits 형식으로 작성해.
```

Claude는 변경 내용을 분석해서 적절한 커밋 메시지를 작성한다.

## 9장. 컨텍스트 관리 전략

### 9.1 컨텍스트 창 이해

Claude Code는 세션 내 대화 내용을 컨텍스트로 유지한다. 세션이 길어질수록:

- 응답이 느려질 수 있음
- 초기 지시를 "잊을" 수 있음
- 토큰 비용이 증가함

### 9.2 컨텍스트 관리 전략

**`/clear` 적극 활용:**
작업 단위가 바뀔 때마다 컨텍스트를 초기화하라. CLAUDE.md가 있으면 핵심 맥락은 자동으로 복원된다.

**`/compact` 사용:**
대화 내용을 요약해서 컨텍스트를 줄이면서도 흐름을 유지할 수 있다.

**세션을 작게 유지:**
하나의 세션에서 너무 많은 작업을 하지 말고, 작업 단위별로 새 세션을 시작하는 것도 좋은 전략이다.

## 10장. 고급 활용 — 자동화와 워크플로우

### 10.1 Claude Code를 CI/CD에 통합

`--print` 플래그로 비대화형(non-interactive) 모드 실행:

```bash
claude --print "코드에서 보안 취약점 분석해줘" > security-report.md
```

### 10.2 커스텀 슬래시 명령어

`~/.claude/commands/` 디렉토리에 마크다운 파일을 만들면 커스텀 `/` 명령어를 정의할 수 있다.

```bash
# ~/.claude/commands/review.md
현재 staged된 변경 사항을 코드 리뷰해줘.
보안, 성능, 가독성 관점에서 피드백을 줘.
```

이후 세션에서 `/review`로 실행 가능.

### 10.3 Hooks — 이벤트 기반 자동화

`~/.claude/settings.json`에 훅을 설정하면 특정 이벤트 발생 시 자동으로 셸 명령을 실행할 수 있다.

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "git add -A && git commit -m 'auto: $(date)'"
          }
        ]
      }
    ]
  }
}
```

파일 변경 시 자동 커밋, 테스트 자동 실행, 알림 발송 등 다양하게 활용할 수 있다.

## 11장. 모범 사례 요약

### Do ✅

- CLAUDE.md에 프로젝트 컨텍스트를 잘 정의해 두기
- 큰 작업은 단계별로 나눠서 체크포인트마다 확인
- 파일 쓰기·명령 실행 전 내용 리뷰
- 작업 전 git 커밋으로 안전망 만들기
- 구체적이고 제약 조건이 명확한 요청 사용

### Don't ❌

- 너무 광범위한 요청을 한 번에 ("전체 앱 만들어줘")
- 컨텍스트가 오염됐는데 계속 진행
- 승인 없이 모든 작업 자동화 (익숙해질 때까지)
- CLAUDE.md에 비밀키나 민감 정보 입력
- Claude의 결과물을 검토 없이 그대로 배포

## 부록. 자주 묻는 질문

**Q. Claude Code와 GitHub Copilot의 차이는?**
Copilot은 에디터 내 코드 완성에 특화되어 있고, Claude Code는 터미널에서 파일 읽기/쓰기/명령 실행까지 가능한 에이전트 방식이다. 용도와 사용 맥락이 다르다.

**Q. 비용은 어떻게 되나?**
Anthropic API 토큰 사용량에 따라 과금된다. `/cost`로 현재 세션 사용량을 확인할 수 있다. 장기 사용이라면 Claude Max 플랜을 고려하라.

**Q. 회사 코드를 Claude에 보내도 안전한가?**
Anthropic의 데이터 처리 정책을 확인하고, 민감한 코드는 사내 데이터 정책에 따라 처리해야 한다. 오프라인 환경이 필요하다면 Anthropic Enterprise 옵션을 검토하라.

**Q. Claude가 틀린 코드를 작성하면?**
Claude는 완벽하지 않다. 테스트를 실행하고, 리뷰를 거치는 것이 필수다. git으로 안전망을 만들어두면 언제든 되돌릴 수 있다.
